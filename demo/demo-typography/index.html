@font-face demo

Setup

To use @font-face, you need to specify at least the following two properties:

  • font-family: the name that will be used as the font-face value for font properties
  • src: resource(s) containing the font data; can be single or a comma-separated list of URLs or names of fonts on the user's computer

Installed Fonts

For installed fonts, all you need to do is specify the font-family, for example font-family: 'Comic Sans MS';

This text is using a font installed on the user's computer!

Locally Hosted Custom Fonts (@font-face)

To use locally hosted fonts, provide the @font-face with a src containing the path to where the font is stored. For example, url('../fonts/Calistoga/Calistoga-Regular.ttf')

Optionally, you can specify the format of the src definition by ending it with format('[FORMAT]')

This text is using a locally hosted font!

Remotely Hosted Custom Fonts (@font-face)

To use remotely hosted fonts, provide the @font-face with a src containing a url to where the font is hosted. For example, url('https://fonts.gstatic.com/s/pacifico/v16/FwZY7-Qmy14u9lezJ-6H6MmBp0u-.woff2')

Or, you can use the stylesheet provided by the remote host. For example, <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Pacifico&display=swap" />
